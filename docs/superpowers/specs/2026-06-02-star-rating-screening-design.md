# 核心层 / 精选层 5★→1★ 上涨概率分级 —— 设计文档

- 日期：2026-06-02
- 分支：feat/liumai-and-combo-screening
- 关联：[[stable_ui.py]] / [[daily_watchlist.py]] / profit_mining 研究基建

## 1. 目标

在「🛡️ 稳定选股」清单的 **核心层** 与 **精选层** 内部，各自把候选股按"未来更容易上涨"
细分成 **5★→1★** 的可信分级标签，帮助用户在同层内优先关注高胜率票。

**硬约束（用户拍板）：星级必须代表"真实样本外验证过的胜率差异"，不是好看的加权分。**
做法是在全历史缠论买点数据集上回测「合成分 → 胜率」关系，定出**固定分数阈值**，
再把阈值套用到每天的清单。

## 2. 现状

`daily_watchlist.py:146-153` 现已有 3 档"精选"标签（离散、非胜率分级）：

| 现档 | 命中条件 | 依据 |
|---|---|---|
| `★★核心` | 1买 + 非陷阱 + 量能金叉 | 1买内 +4.8pt 样本外 |
| `★精选` | 1买 + 非陷阱 | 基础集 +10~12pt（2024-25 OOS） |
| 空（普通） | 其余命中票 | A∪B 基础集 |

本设计在**不破坏现有 3 档层级**的前提下，给"核心"和"精选"两层各加一个层内 1-5★ 子分级。

## 3. 成功口径（用户选择：两个都算，主用 ≥4%，辅看 +10%）

复用现有前向窗口标签，零新增口径：

- **主口径（定星用）**：信号后 30 交易日内 `max(High)/进场价 - 1 ≥ 4%`
  —— 即 `build_features.py` 的 `区间涨跌幅 ≥ WIN_THRESH(=4.0)` / `是否盈利`。
- **辅口径（参考列）**：同窗 `≥ 10%` 大涨 —— `label_window.forward_window_label(win=30, thresh=0.10)`。

两者同出一个 30 日 MFE 窗口，无未来函数泄漏（`label_window.py` 已保证）。

## 4. 打分输入（信号时点特征，严防泄漏）

全部为 `daily_watchlist` 已能算出的现成信号，不新增数据源：

| 特征 | 取值 | 方向 |
|---|---|---|
| 买点类型=1买 | 0/1 | 正 |
| 命中规则=A∪B（双命中） | 0/1 | 正 |
| 资金确认（机构净买） | 0/1 | 正 |
| 中枢底部（极限底/底部回升 ±2窗） | 0/1 | 正 |
| 量能金叉（MA5 上穿 MA60 ±2窗） | 0/1 | 正 |
| 量比 | 分档 ≥1.3 / ≥2 / ≥3 | 正 |
| 获利盘% | 连续，越低越好 | 负（惩罚） |
| 相对强弱（仅 A 类超跌反弹） | 连续，越低越好 | 负 |

## 5. 权重标定（离线一次性回测）

1. 在**全历史缠论买点集**（26249 股 / 9379 正样本，复用 `build_features.py` →
   `signal_features.csv`）上，按主口径 ≥4% 计算每个特征的**样本外胜率增量（提升度）**。
2. 合成分 `score = Σ (权重_k · 特征_k)`，权重取该特征的提升度（正向加、惩罚项减）。
3. 走 walk-forward：训练段（≤2024）定权重与阈值，2025 段独立验证单调性。
4. 产物：`star_thresholds.json`（每层的分数切点 + 各档验证胜率 + 各档 +10% 大涨率）。

## 6. 切档（核心、精选各自独立）+ 诚实降档

- 核心群、精选群**各自**按合成分切档，要求**各档样本外胜率单调递增**（5★最高…1★最低）。
- **诚实降档（用户拍板）**：某层若回测后无法支撑 5 个单调且样本充足的档，就显示该层
  能稳定分出的档数（例如只能稳定到 3 档，则该层 3★ 封顶），并在前台注释说明原因。
  星级数永远对应真实验证过的胜率带，绝不为凑满 5 星制造假精度。
- 阈值固化进 `star_thresholds.json`，每日只做查表，不重新拟合。

## 7. 每日套用（daily_watchlist.py）

对当日每只候选算同一合成分 → 查所在层（核心/精选）阈值表 → 赋 1-N★。
CSV 新增列：

- `星级`：如 `核心★★★★☆` / `精选★★★`（层 + 实心星，封顶档明示）
- `预估胜率`：该档训练验证出的样本外 ≥4% 胜率（带"样本外/参考"字样）
- `大涨率`：该档 +10% 大涨率（辅参考）

## 8. 前台展示（stable_ui.py）

- 清单显示醒目 `星级` 列。
- 排序：**层为主键（核心 > 精选 > 普通），星数层内倒序**（契合"核心 5→1、精选 5→1"）。
- 加注：星级 = 历史样本外胜率分档，非未来保证；仍需结合大盘与基本面人工复核。

## 9. 防过拟合 / 诚实性

- walk-forward 训练/验证分段，2025 段验证单调性后才固化阈值。
- 样本不足层诚实降档（见 §6）。
- `预估胜率` 标注为样本外口径，区别于 241.9 万事件的提升度口径（后者仅锦上添花）。

## 10. 交付范围（用户选择：分两阶段）

  样本不足层正确降档并带注释。
