## 2. 架构

单个独立脚本 `data/profit_mining/surface_l3.py`，纯 pandas，宿主直接运行，不进容器、不依赖 `features_v2.csv`/`features.py`。读全量评分 CSV → 筛 L3 + 算增量 + 护栏 + 排序 → 出独立 L3 榜 CSV + md。

## 3. 数据流

```
各组方案_全部评分_{ts}.csv  (列含: 分组/方案/层级/支持数/盈利覆盖率/非盈利覆盖率/
                            提升度/条件内胜率/基线胜率/胜率增益)
   │ 按"分组"分组处理
   ├─ L2 行 → 子集查找表: frozenset({条件x,条件y}) → 提升度
   ├─ L3 行 → 解析"方案"="a + b + c"(按 " + " 切成 3 个条件名)
   │          → 3 个两两子集 frozenset → 查 L2 提升度
   │          → 最优子集提升度 = max(3 个子集提升度)，记录最优子集
   │          → 增量提升度 = L3提升度 − 最优子集提升度
   ├─ 护栏: 支持数 ≥ 100 且 盈利覆盖率 ≥ 0.20
   └─ 每组按提升度降序取 Top 15 → 合并 → 写 CSV + md
```

## 4. 关键实现细节

### 4.1 子集用集合匹配（核心，避免漏配）
L2 行的"方案"形如 `"x + y"`，其中 `(x,y)` 来自 `itertools.combinations(names, 2)`（names = 信号插入序）；L3 三元组 `"a + b + c"` 来自 `combinations(cand, 3)`（cand 按提升度排序）。两者顺序不同，故两两子集 `(a,b)` 在 L2 里可能存为 `"b + a"`。**解决**：把 L2 每行方案按 `" + "` 切分后转 `frozenset`，建 `dict[frozenset → 提升度]`；L3 的 3 个子集也转 `frozenset` 再查表。

条件显示名不含 `" + "`（用 `>=`/`<=`，如 `"量比>=1.3"`、`"MACD金叉态"`、`"距前高<=10%"`），故按 `" + "` 切分安全。

### 4.2 子集必然存在
三元组支持数 ≥ 100（护栏） ⇒ 每个两两子集支持数 ≥ 三元组支持数 ≥ 100 ≥ 50 ⇒ 一定被保留在 L2 行（L2 已按支持≥50 过滤）。故 3 个子集恒能查到。

### 4.3 提升度 inf 哨兵
现有 `metrics()` 用 `999` 表示 `cl=0`（非盈利覆盖率为 0）的无穷提升度（过拟合标志）。护栏 `覆盖≥0.20` 基本挡掉这类尾部；残留时按 `inf` 语义排序（排顶），增量列中若 L3 或子集任一为 999，增量记为 `inf` 字样不参与数值减法，避免污染。

## 5. 组件

`data/profit_mining/surface_l3.py`：
- `load_scores(path)` → DataFrame（读 CSV，必要列数值化）。
- `build_pair_lift(l2_df)` → `dict[frozenset → 提升度]`（单组的 L2 子集查找表）。
- `split_conditions(plan_str)` → `list[str]`（按 `" + "` 切分方案名）。
- `l3_marginal(l3_row, pair_lift)` → `(最优子集 str, 子集提升度, 增量提升度)`。
- `surface(df, support_min=100, cover_min=0.20, topn=15)` → 榜单 DataFrame（含增量列，按组 Top15）。
- `write_outputs(board, out_dir, ts)` → CSV + md，落 `/home/tdxback/report/`。
- `main()`：默认取 `/home/tdxback/report/` 下最新 `各组方案_全部评分_*.csv`，或命令行传路径。

## 6. 输出（落 `/home/tdxback/report/` 加 `_YYYYMMDD_HHMMSS`）

- `L3独立榜_{ts}.csv`：列 = 分组 / 方案 / 支持数 / 盈利覆盖率 / 条件内胜率 / 基线胜率 / 胜率增益 / 提升度 / **最优两两子集 / 子集提升度 / 增量提升度**。每组 Top15 合并，组内按提升度降序。
- `L3独立榜_对比_{ts}.md`：每组 Top5 摘要，标注**正增量**（第三条真协同）vs **负/零增量**（凑数，三元组只是更窄的 L2）。

## 7. 错误处理与边界

- 全量评分 CSV 不存在 / 无 L3 行 → 明确报错退出，提示先跑 `mine_combos_v2.py`。
- 某组护栏后无 L3 → 该组在榜中缺席，md 注明"无达标 L3"。
- 子集查不到（理论不应发生，见 4.2）→ 该子集提升度记缺失，增量按可得子集算，并计数告警。
- 提升度 999 哨兵按 4.3 处理。

## 8. 测试（TDD，合成小 DataFrame，本地纯 pandas）

- `split_conditions`：`"a + b + c"` → `["a","b","c"]`；含 `>=`/`<=` 的真实名不被误切。
- `build_pair_lift` + `l3_marginal`：构造乱序 pair（L2 存 `"b + a"`，L3 子集 `(a,b)`）断言能匹配；增量 = L3 − 最优子集，断言数值与"最优子集"选取正确。
- `surface` 护栏：支持/覆盖边界值（=100、=0.20 收，99、0.19 弃）；Top15 截断与组内排序；负增量条目保留（不因增量<0 被剔，护栏只看支持+覆盖）。
- inf 哨兵：L3 或子集为 999 时增量记 inf、排序不崩。
