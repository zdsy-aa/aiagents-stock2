# 稳定选股清单新增「可入状态」列 — 设计

日期：2026-06-03
范围：`data/profit_mining/daily_watchlist.py`、`stable_ui.py`（+ 同目录单测）

## 背景与目标

稳定选股清单每行的 `信号日期`（缠论买点日）与 `扫描日期`（批次运行日）本就可能相差数天。
研究纪律规定进场观察窗口为 **±0~±1 交易日最纯（76% 胜率）、±2 提召回精度略降**。
因此用户在某个扫描日看清单时，需要一个直观标注：**这条信号到现在是否还能按原计划进场。**

新增一列 `可入状态`，只做时效/价格状态标注，**不改动任何选股筛选逻辑**。

## 判定输入（扫描时均已具备）

- `信号日期` 在 K 线中的位置 `i`（daily_watchlist 已计算）
- `扫描日期` `sd`
- `买入价` `bp`、`止损价` `stop_loss`、`止盈价` `take_profit`
- 该股已加载的日 K 线 `df`

## 取扫描日收盘价与交易日间隔

- 扫描日收盘：在 `df.index` 上 `searchsorted(扫描日, side="right") - 1` 取 `≤扫描日` 的最后一根
  bar 收盘价（停牌/数据滞后也能落到最近交易日）。记其位置为 `j`。
- 交易日间隔 `gap = j - i`（K 线天然只含交易日，位置差即交易日差，恒 `≥0`）。
- 越界保护：`j < i`（理论不应发生）或扫描日早于所有 bar → `gap` 视为无效，状态置 `—`。

## 标签逻辑（按优先级，先命中先定）

| 优先级 | 条件 | 标签 |
|---|---|---|
| 1 | 扫描日收盘 < 止损价 | `已破止损` |
| 2 | 扫描日收盘 ≥ 止盈价 | `已止盈` |
| 3 | 扫描日收盘 > 买入价 × 1.05 | `已涨过` |
| 4 | gap > 2 交易日 | `已过窗` |
| 5 | gap == 2 交易日 | `尾窗` |
| 6 | gap ≤ 1 交易日 | `可入` |

价格类（破止损/止盈/涨过）优先于窗口类——价格跑掉时进场前提已不在，即便仍在 ±2 窗口内也不该追。
涨过阈值固定 +5%（抄底逻辑下走得越远性价比越差，取较紧口径）。

## 纯函数（可测试单元）

在 `daily_watchlist.py` 顶部定义：

```python
def entry_status(gap, close, buy, stop, tp, premium=0.05):
    """返回可入状态标签。gap=信号到扫描日的交易日数(>=0)；close=扫描日收盘；
    buy/stop/tp=买入/止损/止盈价。gap<0 或入参缺失→ '—'。"""
```

- 输入仅为标量（int/float），无 pandas 依赖，便于单测。
- `close`/`buy` 为 None 或 `gap < 0` → 返回 `"—"`。

## 落地

- **daily_watchlist.py**：在算出 `bp/stop_loss/take_profit` 后，定位 `j`、算 `gap`、调用
  `entry_status(...)`，把结果写入每行字典键 `可入状态`，并加入 `cols`（放在 `优先级` 之后）。
  历史重跑（带 scan_date 参数）也能按当日收盘正确还原。
- **stable_ui.py**：把 `可入状态` 加入前置列组（与 星级/预估胜率/大涨率 并列，最显眼）；
  旧 CSV 无此列 → 填 `—`；加一行图例说明 6 种取值。

（gap<0、close=None、优先级覆盖：破止损早于已过窗）各一个用例，纯标准库 / pytest，

## 部署影响

- daily_watchlist 改动：容器即时生效（下次 20:00 批量或手动刷新即带新列）。
- stable_ui 改动：需重建镜像才对用户可见（与既往 UI 改动一致）。
- 既有历史 CSV 不回填，UI 优雅显示 `—`。
